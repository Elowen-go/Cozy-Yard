### 开发优先级

开发顺序固定为：

1. **M0：可操作原型**：玩家能进入庭院、移动、采集、种植、收获并升级篝火。
2. **M1：核心循环**：补齐 3 块菜圃、基础资源栏、植物图鉴和第一道菜谱。
3. **M2：内容扩展**：加入省份地图、四川探索和精力系统。
4. **M3：中后期系统**：加入旅行商人、杂交、堆肥和更多烹饪内容。
5. **M4：内容完成与打磨**：扩展至完整图鉴、菜谱、省份、结局和无尽模式。

在 M0 和 M1 验收前，不制作 120 种植物、80 道菜谱或 12 省份的完整内容。

### M0 原型验收标准

M0 完成必须满足以下条件：

- 启动项目后可以进入庭院主场景。
- 玩家可以使用 WASD 或方向键移动。
- 场景中有 1 个可交互资源点，点击后获得木头，并有可见反馈。
- 场景中有 1 块菜圃，可以完成“开垦→播种→生长→收获”。
- 至少有 1 种作物，生长状态至少区分种子、幼苗、成熟。
- 场景中有 1 个篝火，可以消耗木头升级一次，并影响作物生长时间。
- 资源数值显示在界面上，重新运行不会出现脚本报错。

M0 只要求一条完整可玩的循环，不要求最终美术、复杂动画或完整数据量。

### 每个功能的交付要求

新增功能必须同时具备：

1. 场景或脚本中的实际实现。
2. 玩家可以触发的最小交互流程。
3. 至少一种明确的视觉或文字反馈。
4. 一个可重复执行的验收步骤。
5. 如果涉及存档，需验证退出后重新进入仍能恢复状态。

## 一、项目概述

### 1.1 游戏定位

《闲庭》是一款以**中国植物收集**和**中华美食图鉴**为核心的治愈系种菜放置游戏。玩家在自家庭院中种植作物、收集图鉴、解锁菜谱，同时通过探索中国各省份地图，发现更多稀有植物与食材。

**一句话概括**：一部可以亲手种植的中华风物志，一场走遍中国地图的治愈之旅。

### 1.2 核心特色

| 特色 | 说明 |
| :--- | :--- |
| **种菜收集** | 种植120种中国代表性植物，每收获一种即点亮图鉴 |
| **省份探索** | 12个美食省份地图探索，发现区域特色植物与食材 |
| **美食图鉴** | 解锁80道中华美食菜谱，附《舌尖》风格文艺文案 |
| **治愈放置** | 无惩罚、无倒计时，任何时候回来庭院都在静静生长 |
| **视觉成长** | 篝火从火星到烈焰，庭院从荒芜到繁盛，所见即所得 |

### 1.3 技术参数

| 项目 | 规格 |
| :--- | :--- |
| 游戏引擎 | Godot 4.x（GDScript） |
| 画面风格 | 像素风（主角/庭院）+ 像素水彩风（探索场景） |
| 视角 | 2D俯视角（种菜主场景）/ 2D横版（探索场景） |
| 目标平台 | PC（itch.io首发）/ 移动端（后期适配） |
| 存档方式 | 本地JSON存档 |

## 三、六大核心系统详解

### 3.1 种植系统

**功能**：玩家通过种植、等待、收获获得植物果实，解锁图鉴。

**操作流程**：
1. 点击闲置菜圃 → 消耗木头开垦
2. 选择种子（从背包中）→ 播种
3. 等待作物生长（受篝火等级加成）
4. 成熟后点击“收获” → 获得果实 → 图鉴进度+1

**菜圃规格**：
- 初始：3块
- 最大：9块（3×3网格）
- 扩展方式：消耗木头逐步解锁

**生长时间（篝火Lv.1基准）**：

| 稀有度 | 代表植物 | 生长时间 |
| :--- | :--- | :--- |
| 普通（绿色） | 小麦、白菜、葱 | 3分钟 |
| 稀有（蓝色） | 花椒、香菇、莲藕 | 15分钟 |
| 珍贵（紫色） | 松茸、荔枝、龙井 | 30分钟 |
| 传奇（金色） | 人参、灵芝、雪莲 | 2小时 |

**视觉反馈**：每个作物有3个生长阶段（种子→幼苗→成熟），像素动画逐级变化。

---

### 3.2 资源挂机系统

**功能**：玩家通过点击和自动挂机获取基础资源（木头、草药、金币），用于升级和兑换。

**资源类型**：

| 资源 | 产出方式 | 主要用途 |
| :--- | :--- | :--- |
| **木头** | 点击树木（冷却3秒）+ 自动挂机 | 开垦菜圃、升级篝火、建造设施 |
| **草药** | 点击草丛（冷却5秒）+ 自动挂机 | 合成肥料、购买稀有种子 |
| **金币** | 点击宝箱（冷却30秒）+ 自动挂机 | 旅行商人购买、购买种子 |

**自动挂机机制**：
- 小骑士每隔数秒自动挥剑
- 产出量 = 基础值 ×（1 + 篝火等级 × 10%）
- 离线也持续产出（最多累计8小时）

---

### 3.3 篝火升级系统

**功能**：全局加速核心，投入资源升级篝火，提升所有生长和产出效率。

**升级消耗**：
- 每级消耗：木头（基础）+ 草药（中后期）+ 特殊果实（后期）
- 消耗量递增（Lv.1→2: 10木；Lv.9→10: 200木+50草药+5稀有果）

**升级效果**：
- 种植生长速度 +5%/级
- 挂机产出 +10%/级
- 解锁新功能（如堆肥桶、昼夜循环）

**视觉反馈**（每2级变化一次）：
- Lv.1-2：小火苗
- Lv.3-4：明亮火焰
- Lv.5-6：火焰带橙色余烬
- Lv.7-8：火焰带彩色光晕
- Lv.9-10：彩虹色“永恒之焰”

---

### 3.4 图鉴系统

**功能**：记录所有已收集植物，提供收集目标和省份进度。

**植物分类**（120种）：

| 类别 | 数量 | 代表植物 |
| :--- | :--- | :--- |
| 粮食作物 | 30种 | 水稻、小麦、玉米、高粱、小米、大豆、红薯、土豆、花生…… |
| 蔬菜/香料 | 50种 | 辣椒、花椒、姜、蒜、葱、白菜、萝卜、茄子、黄瓜、冬瓜、莲藕、竹笋、菌菇类…… |
| 水果/经济作物 | 40种 | 荔枝、龙眼、柑橘、苹果、梨、桃、李、杏、枣、柿子、葡萄、西瓜、茶叶、甘蔗…… |

**稀有度分级**：

| 等级 | 颜色 | 数量 | 获取方式 |
| :--- | :--- | :--- | :--- |
| 普通 | 绿色 | ~50种 | 商店直接购买 |
| 稀有 | 蓝色 | ~30种 | 探索获得 / 特定条件 |
| 珍贵 | 紫色 | ~20种 | 杂交变异 / 旅行商人 |
| 传奇 | 金色 | ~12种 | 省份全收集奖励 / 极稀有探索 |
| 神话 | 红色 | ~6种 | 集齐某省全部植物后奖励 |
| 传说 | 彩虹 | 2种 | 全图鉴100%解锁（人参、灵芝） |

**省份分组**（12省）：

四川、广东、江苏、浙江、山东、福建、云南、陕西、湖南、湖北、安徽、东北（黑吉辽合并）

**图鉴界面布局**：
- 左侧：12个省份标签页
- 右侧：植物卡片网格（彩色=已收集，灰色剪影=未收集）
- 点击卡片 → 弹出详情页（大图+介绍+可制作菜谱）

---

### 3.5 厨房与菜谱系统

**功能**：玩家通过组合食材解锁中华美食菜谱，完成第二收集目标。

**菜谱构成**（80道）：

| 解锁方式 | 占比 | 数量 | 举例 |
| :--- | :--- | :--- | :--- |
| 自然解锁 | 40% | 32道 | 收获小麦+猪肉 → 自动解锁“饺子” |
| 合成烹饪 | 40% | 32道 | 手动组合辣椒+花椒+牛肉 → “水煮牛肉” |
| 事件解锁 | 20% | 16道 | 旅行商人出售“菜谱残页” |

**烹饪操作流程**：
1. 进入厨房界面（点击灶台）
2. 左侧显示已收获食材仓库
3. 手动拖拽食材到锅具槽位（最多4格）
4. 点击“开始烹饪”
5. 正确组合 → 解锁菜谱 + 弹出美食卡
6. 错误组合 → 提示“味道不对”，返还80%食材

**菜谱详情页内容**：
- 像素美食插画
- 菜名 + 所属城市/省份
- 食材配方清单
- 《舌尖》风格文艺文案（约80字）
- 熟练度星级（可多次烹饪提升）

---

### 3.6 探索系统

**功能**：通过省份地图探索发现新植物、稀有食材和见闻日志，是图鉴收集的核心途径。

**探索模式对比**：

| 模式 | 消耗 | 产出 | 适用场景 |
| :--- | :--- | :--- | :--- |
| 自动探索 | 免费 | 基础资源（木头x1/金币x1），不产出新植物 | 挂机补充资源 |
| 手动探索 | 精力x1 | 必定获得新植物/稀有食材 + 见闻日志 | 首次探索/解锁图鉴 |

**探索流程**：
1. 打开地图界面 → 点击已解锁省份
2. 该省份弹出3个探索场景卡片（山林/市集/茶园等）
3. 点击一个场景 → 点击“出发探索”
4. 进入场景画面（像素水彩风静态图 + 小骑士自动走动）
5. 观看探索动画（2-3分钟，可切换回闲庭但奖励取消）
6. 小骑士带回奖励 → 弹窗显示获得物品 + 见闻日志
7. 返回闲庭

**精力系统**：
- 上限：5点
- 恢复：每30分钟恢复1点（离线也恢复）
- 消耗：每次手动探索消耗1点

**探索场景类型**（每省3个）：

| 场景类型 | 产出倾向 | 示例（四川） |
| :--- | :--- | :--- |
| 山林 | 野生植物/稀有种子 | 野生花椒种子 |
| 市集 | 特殊食材/菜谱残页 | 郫县豆瓣酱 |
| 湖泊/湿地 | 水生植物 | 莲藕 |
| 古寺/茶园 | 文化植物+文案碎片 | 蒙顶甘露茶叶 |
| 农田 | 基础作物（量大） | 水稻x5 |

**省份解锁条件**：
- 初始解锁：四川、广东、山东
- 后续解锁：集齐相邻省份6种植物后解锁下一个

**见闻日志**：每次探索获得一条，自动收录在图鉴“省份笔记”中，集齐一省全部见闻解锁“风物志”成就。

### 5.1 Godot 节点结构

```
Main.tscn (主场景)

│   │   ├── Sprite2D (动画)
│   │   └── CollisionShape2D
│   ├── 菜圃 (Area2D × 9)
│   │   ├── Sprite2D (状态: 荒地/种子/幼苗/成熟)
│   │   └── CollisionShape2D (点击检测)
│   ├── 资源点 (树木/草丛/宝箱)
│   │   ├── Area2D
│   │   ├── Sprite2D
│   │   └── Timer (冷却)
│   ├── 篝火
│   │   ├── Sprite2D (多帧动画)
│   │   └── Area2D (点击升级)
│   └── 装饰物 (Node2D × N)
├── CanvasLayer (UI层)
│   ├── 资源栏 (木头/草药/金币)

│   ├── 探索弹窗 (场景画面)
│   └── 提示系统
└── 自动加载 (AutoLoad)

```

| `Exploration.gd` | 探索系统逻辑 | `start_explore()`, `get_reward()`, `_on_timer_timeout()` |

## 六、开发路线图（分阶段实施）

### 6.1 开发总路径

我们按照“先做一条完整小循环，再扩内容”的顺序开发，不并行制作所有系统。每个阶段完成后先验收，验收通过才进入下一阶段。

| 阶段 | 目标 | 阶段结束时玩家能做什么 |
| :--- | :--- | :--- |
| **M0 原型闭环** | 证明种菜玩法可玩 | 移动、采集木头、种植 1 种作物、收获、升级篝火 |
| **M1 家园核心循环** | 把资源、种植、图鉴、烹饪连起来 | 管理 3 块菜圃，收获 5 种植物，解锁图鉴并制作 1 道菜 |
| **M2 四川探索** | 验证第二场景和探索奖励 | 打开地图，进入四川的 3 个场景，消耗精力并带回收集物 |
| **M3 内容与成长** | 增加重复游玩价值 | 使用肥料、杂交作物、旅行商人和更复杂的菜谱 |
| **M4 完整内容与发布** | 完成目标内容并打磨体验 | 完成主要图鉴、菜谱、省份、结局和无尽模式 |

### 6.2 统一完成定义

任何一个阶段只有同时满足以下条件，才算完成：

- [ ] 从项目启动到该阶段主流程可以完整走通。
- [ ] 阶段内的核心功能可以由玩家实际操作，而不是只有占位节点或打印日志。
- [ ] 关键状态有视觉、文字或动画反馈。
- [ ] 正常操作和错误操作都有明确结果，例如资源不足时不能继续消耗。
- [ ] 重新启动项目后，阶段要求保存的数据仍然正确。
- [ ] Godot 调试器没有新增脚本错误或持续刷屏的运行错误。
- [ ] 按阶段验收步骤重复测试至少两次，结果一致。
- [ ] 文档中的“已实现”列表、实际场景树和脚本文件已经同步。

### 6.3 M0：原型闭环

**目标**：用最少内容验证“采集资源→种植→等待→收获→升级”的核心循环。

**开发顺序**：

1. 创建 `Player` 场景和移动脚本，接入主场景。
2. 添加一个资源点，点击后获得木头，并显示采集反馈。
3. 添加一块菜圃，定义空闲、已开垦、已播种、生长中、成熟五种状态。
4. 添加一种作物，实现播种、生长计时和收获。
5. 添加篝火，实现消耗木头升级一次，并让等级影响生长时间。
6. 添加最小资源栏，显示木头和篝火等级。
7. 增加 M0 所需的本地 JSON 存档。

**阶段产出**：

- `Player` 场景和移动脚本。
- `Crop` 或菜圃场景和作物状态逻辑。
- `Campfire` 场景和升级逻辑。
- 一个资源点场景。
- 一个最小 UI 场景或 CanvasLayer。
- 一个全局状态/存档脚本。

**玩家可见效果**：

玩家启动游戏后，可以在草地上移动；点击资源点获得木头；用木头开垦菜圃并播种；等待后作物成熟；收获后资源增加；投入木头升级篝火后，下一轮作物生长更快。

**M0 验收**：

- [x] WASD 或方向键可以移动玩家，玩家不会跑出地图。
- [x] 点击资源点后木头增加，资源点有一次点击反馈。
- [x] 没有木头时不能开垦或升级，界面会显示原因。
- [x] 一块菜圃能完成开垦、播种、生长、成熟、收获。
- [x] 收获只能在成熟状态进行，重复点击不会重复领取。
- [x] 篝火升级后等级增加，生长时间确实发生变化。
- [x] 退出并重新启动后，木头、篝火等级和作物状态能够恢复。

**进入 M1 的门槛**：M0 主流程可以连续完成两次，且没有脚本报错、状态丢失或重复领取奖励的问题。

### 6.4 M1：家园核心循环

**目标**：把家园场景变成一个可以持续游玩的最小版本。

**开发顺序**：

1. 将菜圃扩展为 3 块，统一使用同一套菜圃状态逻辑。
2. 增加木头、草药、金币三类资源，并明确每类资源的来源和用途。
3. 增加 5 种基础植物数据，至少区分名称、生长时间、收获物和稀有度。
4. 增加植物图鉴，显示已收集和未收集状态。
5. 增加第一版厨房，支持 1 道菜的材料检查和制作。
6. 增加基础提示、按钮禁用和资源不足反馈。
7. 完善存档，覆盖菜圃、资源、植物图鉴和菜谱。

**玩家可见效果**：

玩家可以同时管理 3 块菜圃，选择不同作物安排种植；收获新植物后图鉴点亮；材料足够时进入厨房制作第一道菜；资源和进度在重新启动后仍然保留。

**M1 验收**：

- [ ] 3 块菜圃可以独立种植，互不覆盖状态。
- [ ] 5 种植物至少有 3 种不同生长时间或收获结果。
- [ ] 图鉴能正确区分已收集和未收集植物。
- [ ] 材料不足时不能制作菜谱，材料足够时只扣除一次。
- [ ] 资源栏、图鉴和厨房显示的是同一份全局数据。
- [ ] 存档覆盖 3 块菜圃、资源、图鉴和菜谱状态。
- [ ] 连续游玩 15 分钟没有重复奖励、负数资源或状态错乱。

**进入 M2 的门槛**：M1 能够支持一次完整的“种植多块菜圃→收获植物→点亮图鉴→制作菜谱”流程，并通过存档恢复测试。

### 6.5 M2：四川探索

**目标**：验证家园之外的地图、探索场景、精力和奖励循环。

**开发顺序**：

1. 创建中国地图界面和省份选择状态，先只开放四川。
2. 创建四川的 3 个探索场景：山林、市集、茶园或农田。
3. 创建探索流程：选择场景、消耗精力、播放过程、结算奖励、返回闲庭。
4. 增加探索奖励池，至少包含 1 种植物、1 种食材和 1 条见闻日志。
5. 增加精力上限、消耗、恢复和不足提示。
6. 把探索奖励接入植物图鉴、菜谱材料和省份进度。

**玩家可见效果**：

玩家在闲庭点击地图按钮，选择四川的一个场景并消耗 1 点精力；探索结束后获得新植物、食材或日志；返回闲庭后奖励已经进入对应系统。

**M2 验收**：

- [ ] 地图界面只开放设计中规定的省份，未开放省份不可进入。
- [ ] 四川 3 个场景可以分别进入，场景名称和产出倾向不同。
- [ ] 精力不足时不能开始手动探索。
- [ ] 探索完成只结算一次奖励，重复返回不会重复领取。
- [ ] 植物、食材和日志分别进入图鉴、资源或日志列表。
- [ ] 闲庭和地图之间可以双向返回，返回后原场景状态不丢失。
- [ ] 探索相关状态可以保存和恢复。

**进入 M3 的门槛**：至少完成 3 次探索，奖励、精力和返回流程全部稳定，且不会重复结算。

### 6.6 M3：内容与成长

**目标**：增加长期成长和重复游玩，但仍然优先保证规则清晰、数据可维护。

**开发顺序**：

1. 增加肥料和堆肥桶，明确产出、消耗和加速规则。
2. 增加杂交或变异作物，先实现 1 种明确变异规则。
3. 增加旅行商人和有限库存刷新。
4. 将厨房扩展为多材料配方和错误组合反馈。
5. 增加昼夜或庭院成长视觉变化，确保视觉变化对应真实状态。
6. 抽离植物、菜谱、商店和探索奖励为独立数据文件。

**M3 验收**：

- [ ] 肥料、杂交、商人和复杂菜谱都有明确的输入、输出和失败规则。
- [ ] 商店刷新不会造成重复购买或负数库存。
- [ ] 变异结果可保存、可在图鉴中查看。
- [ ] 数据文件错误时能给出可定位的错误，而不是静默产生错误奖励。
- [ ] 核心循环连续运行 30 分钟，资源和存档状态保持一致。

**进入 M4 的门槛**：M3 的新增系统都已通过独立验收，且不会破坏 M1 和 M2 的基础流程。

### 6.7 M4：完整内容与发布

**目标**：完成正式内容规模，统一视觉、交互、性能和存档体验。

**开发顺序**：

1. 扩展植物数据至 120 种，逐项核对来源、稀有度、产地和获取方式。
2. 扩展菜谱数据至 80 道，逐项核对材料、解锁方式和文案。
3. 扩展至 12 个省份和 36 个探索场景。
4. 完成结局、无尽模式和全收集奖励。
5. 完成新手引导、设置、音效、动画、错误提示和操作反馈。
6. 做完整存档兼容、异常退出、长时间运行和发布包测试。

**M4 验收**：

- [ ] 目标数量与数据实际数量一致：120 种植物、80 道菜谱、12 个省份、36 个探索场景。
- [ ] 所有收集物都有获取路径，不能出现无法获得或无法完成的条目。
- [ ] 新玩家不看开发者说明也能完成第一次种植和第一次探索。
- [ ] 存档升级、损坏提示和备份恢复流程可用。
- [ ] 发布包在目标平台启动、保存、读取和退出正常。
- [ ] 完成一次从新游戏到结局的完整回归测试。

### 6.8 每次开发任务的固定流程

以后每个功能都按以下顺序执行：

1. 从当前阶段任务清单中选择一个最小功能。
2. 先确定场景节点、脚本职责、数据输入和验收条件。
3. 实现最小可操作版本，不提前加入后续阶段内容。
4. 在 Godot 中运行并验证正常路径、资源不足和重复操作。
5. 检查调试器错误、存档恢复和场景重新打开结果。
6. 更新本文档的任务勾选、当前状态和实际文件路径。
